Làm chủ ngôn ngữ SQL: Từ cơ bản đến nâng cao

Ngày cập nhật: 13/03/2026 Chuyên mục: Database (Cơ sở dữ liệu)

SQL (Structured Query Language) là ngôn ngữ tiêu chuẩn để tương tác với các hệ quản trị cơ sở dữ liệu quan hệ. Dưới đây là tóm tắt các cú pháp quan trọng nhất.

I. Cấu trúc câu lệnh truy vấn cơ bản

Một câu lệnh SQL hoàn chỉnh thường tuân theo thứ tự sau:

UPDATE <bảng> SET <cột>=<giá trị> WHERE <điều kiện>;
INSERT INTO <bảng> (<cột>) VALUES (<giá trị>);
DELETE FROM <bảng> WHERE <điều kiện>;
SELECT [tính chất] <danh sách thuộc tính>
FROM <danh sách table hoặc query/view [as alias]>
[WHERE <điều kiện lọc mẫu tin>]
[GROUP BY <nhóm dữ liệu>]
[HAVING <điều kiện sau khi nhóm>]
[ORDER BY <thuộc tính sắp xếp> ASC | DESC]
Từ khóa Ý nghĩa
SELECTXác định các cột cần hiển thị trong kết quả.
UPDATECập nhật giá trị của các cột trong bảng.
INSERT INTOChèn dữ liệu mới vào bảng.
DELETE FROMXóa các bản ghi khỏi bảng.
FROMXác định các bảng cần lấy dữ liệu.
WHERELọc các bản ghi thỏa mãn điều kiện nhất định.
ORDER BYSắp xếp kết quả, mặc định là tăng dần.
GROUP BYNhóm dữ liệu có cùng giá trị ở một hoặc nhiều cột.
HAVINGLọc các nhóm dữ liệu sau khi đã GROUP BY.

II. Các phép toán so sánh và điều kiện đặc biệt

  • Tìm kiếm đơn giản: Kết hợp nhiều điều kiện bằng AND hoặc OR.
  • IS NULL / IS NOT NULL: Kiểm tra giá trị trống hoặc có dữ liệu.
  • IN / NOT IN: Kiểm tra xem một giá trị có nằm trong một tập hợp cho trước hay không.
  • LIKE: Dùng cho dữ liệu chuỗi với ký tự đại diện như %.
  • DATEDIFF(): Tính khoảng cách ngày giữa hai biểu thức.
Mẹo nhỏ: Khi so sánh chuỗi, hãy ưu tiên sử dụng LIKE để tránh các lỗi so sánh chính xác tuyệt đối không cần thiết.

III. Phép kết (JOIN) - Truy vấn từ nhiều bảng

Khi dữ liệu nằm ở nhiều bảng khác nhau, chúng ta cần sử dụng phép JOIN để kết hợp các bảng lại với nhau dựa trên một điều kiện liên kết.

1. INNER JOIN (JOIN)

INNER JOIN trả về các dòng có giá trị khớp ở cả hai bảng. 

SELECT *
FROM BangA
INNER JOIN BangB
ON BangA.id = BangB.id;

2. LEFT JOIN (LEFT OUTER JOIN)

LEFT JOIN trả về tất cả các bản ghi từ bảng bên trái và các bản ghi khớp từ bảng bên phải. 

SELECT *
FROM BangA
LEFT JOIN BangB
ON BangA.id = BangB.id;

3. RIGHT JOIN (RIGHT OUTER JOIN)

RIGHT JOIN trả về tất cả các bản ghi từ bảng bên phải và các bản ghi khớp từ bảng bên trái. 

SELECT *
FROM BangA
RIGHT JOIN BangB
ON BangA.id = BangB.id;

4. Ví dụ minh họa

Giả sử có hai bảng: SinhVien (id, ten, ma_lop) và Lop (id, ten_lop). 

SELECT SinhVien.ten, Lop.ten_lop
FROM SinhVien
INNER JOIN Lop
ON SinhVien.ma_lop = Lop.id;

IV. SQL lồng (Subquery)

Subquery là truy vấn lồng bên trong một truy vấn khác, thường dùng để lọc dữ liệu hoặc tính toán trung gian.

1. Subquery trong mệnh đề WHERE

SELECT ten, luong
FROM NhanVien
WHERE luong > (
    SELECT AVG(luong)
    FROM NhanVien
);

2. Subquery với toán tử IN

SELECT ten_sinh_vien
FROM SinhVien
WHERE ma_lop IN (
    SELECT id
    FROM Lop
    WHERE ten_lop LIKE 'CNTT%'
);

3. Subquery trong mệnh đề FROM

SELECT tmp.ma_lop, tmp.so_luong
FROM (
    SELECT ma_lop, COUNT(*) AS so_luong
    FROM SinhVien
    GROUP BY ma_lop
) AS tmp
WHERE tmp.so_luong > 30;
Lưu ý: Nếu subquery trả về nhiều dòng, hãy dùng IN, EXISTS hoặc so sánh phù hợp; tránh dùng toán tử =.
