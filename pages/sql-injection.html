Làm chủ ngôn ngữ SQL: Từ cơ bản đến nâng cao

Ngày cập nhật: 13/03/2026 Chuyên mục: Database (Cơ sở dữ liệu)

SQL (Structured Query Language) là ngôn ngữ tiêu chuẩn để tương tác với các hệ quản trị cơ sở dữ liệu quan hệ. Dưới đây là tóm tắt các cú pháp quan trọng nhất.

I. Cấu trúc câu lệnh truy vấn cơ bản

Một câu lệnh SQL hoàn chỉnh thường tuân theo thứ tự sau:

SELECT [tính chất] <danh sách thuộc tính>
FROM <danh sách table hoặc query/view [as alias]>
[WHERE <điều kiện lọc mẫu tin>]
[GROUP BY <nhóm dữ liệu>]
[HAVING <điều kiện sau khi nhóm>]
[ORDER BY <thuộc tính sắp xếp> ASC | DESC]
Từ khóa Ý nghĩa
SELECTXác định các cột cần hiển thị trong kết quả.
FROMXác định các bảng (tables) cần lấy dữ liệu.
WHERELọc các bản ghi thỏa mãn điều kiện nhất định.
ORDER BYSắp xếp kết quả (Mặc định là ASC - tăng dần).

II. Các phép toán so sánh và điều kiện đặc biệt

  • Tìm kiếm đơn giản: Kết hợp nhiều điều kiện bằng AND hoặc OR.
  • IS NULL / IS NOT NULL: Kiểm tra giá trị trống (NULL) hoặc có dữ liệu.
  • IN / NOT IN: Kiểm tra xem một giá trị có nằm trong một tập hợp cho trước hay không.
  • Phép LIKE: Dùng cho dữ liệu chuỗi (String). Thay vì dùng dấu =, dùng LIKE kết hợp với ký tự đại diện (ví dụ %) để tìm kiếm linh hoạt hơn.
  • DATEDIFF(): Tính khoảng cách ngày giữa hai biểu thức expr1 - expr2.
Mẹo nhỏ: Khi so sánh chuỗi, hãy luôn ưu tiên sử dụng LIKE để tránh các lỗi so sánh chính xác tuyệt đối không cần thiết.

III. Phép Kết (Join) - Truy vấn từ nhiều bảng

Khi dữ liệu nằm ở nhiều bảng khác nhau, chúng ta cần thực hiện phép kết tại mệnh đề FROM và thiết lập điều kiện kết tại WHERE.

Nguyên tắc kết dựa trên khóa:

Thông thường, chúng ta kết bảng dựa trên mối quan hệ Khóa chính - Khóa ngoại:

SELECT *
FROM Bang_A, Bang_B
WHERE Bang_A.Khoa_Chinh = Bang_B.Khoa_Ngoai;

Ví dụ thực tế:

Lấy tên sinh viên và tên lớp mà sinh viên đó đang học:

SELECT SINHVIEN.HoTen, LOP.TenLop
FROM SINHVIEN, LOP
WHERE SINHVIEN.MaLop = LOP.MaLop;
